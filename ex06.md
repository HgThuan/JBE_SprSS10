# Tổng hợp xử lý Naming và Formatting

### Tình huống
Một đồng nghiệp thiết kế API: `POST /api/doctor/update-info?id=10`

---

## 1. Các điểm sai quy tắc RESTful trong API trên
API trên vi phạm các nguyên tắc thiết kế RESTful cơ bản sau:
1. **Sử dụng sai HTTP Method:** `POST` thường được dùng để tạo mới một tài nguyên (Create). Để cập nhật (Update) tài nguyên đã tồn tại, RESTful quy định phải sử dụng `PUT` (cập nhật toàn phần) hoặc `PATCH` (cập nhật một phần).
2. **Sử dụng động từ trong URL:** REST yêu cầu URL chỉ chứa danh từ đại diện cho tài nguyên, không chứa hành động. Đường dẫn `/update-info` là một động từ, vi phạm chuẩn RESTful.
3. **Danh từ số ít:** Tài nguyên nên được đặt tên dưới dạng số nhiều để đại diện cho một tập hợp. URL dùng `/doctor` (số ít) thay vì `/doctors` (số nhiều).
4. **Sử dụng Query Parameter thay vì Path Variable:** Khi tác động trực tiếp lên một tài nguyên cụ thể đã biết ID (để sửa/xóa/xem chi tiết), ID đó phải là một phần của đường dẫn (Path Variable), ví dụ `/doctors/10`. Không dùng Query Param `?id=10` cho mục đích định danh tài nguyên chính.

---

## 2. Viết lại URL chuẩn RESTful
- **HTTP Method:** `PUT` (hoặc `PATCH` nếu chỉ cập nhật 1 vài trường).
- **URL (Path & Param):** `/api/v1/doctors/10` (Thêm versioning `/v1/` để dễ quản lý, đổi thành số nhiều `doctors`, đưa số `10` làm Path Variable và xóa bỏ động từ cập nhật).

---

## 3. Thiết kế phần Body (JSON) cập nhật
Nội dung Request Body tuân thủ chuẩn đặt tên `camelCase`:
```json
{
  "firstName": "Hoang",
  "lastName": "Thuan",
  "specializationName": "Khoa Tim Mạch",
  "phoneNumber": "0987654321",
  "yearsOfExperience": 12,
  "emailAddress": "doctor@hospital.com"
}
```

---

## 4. Response mẫu (Đồng nhất cấu trúc Wrapper)

### Khi cập nhật thành công (Status 200 OK)
```json
{
  "status": "success",
  "code": 200,
  "data": {
    "id": 10,
    "firstName": "Hoang",
    "lastName": "Thuan",
    "specializationName": "Khoa Tim Mạch",
    "phoneNumber": "0987654321",
    "yearsOfExperience": 12,
    "emailAddress": "doctor@hospital.com"
  }
}
```

### Khi ID bác sĩ không tồn tại (Status 404 Not Found)
Sử dụng lại cấu trúc `GlobalExceptionHandler` đã định nghĩa ở bài trước:
```json
{
  "timestamp": "2026-07-22T09:20:00.123456",
  "status": 404,
  "error": "Not Found",
  "message": "Không tìm thấy bác sĩ với ID: 10",
  "path": "/api/v1/doctors/10"
}
```
